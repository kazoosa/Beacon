## What this site is for

This is the engineering source of truth for Beacon. New engineers land here
on day one and shouldn't need to read the codebase to know what we built,
why we built it that way, and how to operate it. Existing engineers come
back for runbooks and ADRs.

The content is hand-written; the test↔feature traceability matrix is
auto-generated from `vitest` describe blocks at build time so it stays
honest.

## Conventions

- **Code paths** are linked relative to repo root: `apps/backend/src/...`
- **Commit hashes** in ADRs and epics link to GitHub.
- **External services** (Render, Vercel, Neon, Upstash, Tradier, SnapTrade)
  are introduced in [Architecture → External services](/architecture/external-services/)
  before they're referenced anywhere else.
- **TODOs / known gaps** are explicit. If we haven't decided something,
  the doc says "open question" — it doesn't pretend.
